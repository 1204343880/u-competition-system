### 发现的断点及修复

**断点 1：Redis 未运行导致后端启动失败**
- 严重度：阻塞
- 现象：后端启动时报 Redis 连接失败
- 解决：从 `D:\REDIS\Redis-7.4.3-Windows-x64-msys2\` 启动 Redis

**断点 2：RabbitMQ 未安装**
- 严重度：阻塞
- 现象：系统中未找到 RabbitMQ，所有通知功能不可用
- 解决：用户在 Docker 上启动 RabbitMQ 容器

**断点 3：通知消费者反序列化失败**
- 严重度：阻塞（代码 Bug）
- 现象：RabbitMQ 消息消费时 Jackson 反序列化报错
- 涉及文件：`innotrack-framework/.../mq/RabbitMqConfig.java`
- 修复：添加 `Jackson2JsonMessageConverter` 配置 + `rabbitListenerContainerFactory` + `RabbitTemplate#setMessageConverter`

**断点 4：知识库审核通知 userId 获取错误**
- 严重度：阻塞（代码 Bug）
- 现象：`Long.valueOf(createBy)` 报错，createBy 是用户名字符串而非数字 ID
- 涉及文件：`innotrack-admin/.../controller/competition/KnowledgeAuditController.java` 第 160/192 行
- 修复：改为 `userService.selectUserByUserName(createBy)` 再取 `getUserId()`

**断点 5：nginx 前端挂载路径错误**
- 严重度：阻塞（配置错误）
- 现象：前端页面全部 404
- 涉及文件：`docker-compose.yml`
- 修复：挂载路径从 `./ruoyi-ui/dist` 改为 `./innotrack-ui/dist`

**断点 6：nginx.conf 内容错误**
- 严重度：阻塞（配置错误）
- 现象：nginx 配置内容不正确，SPA fallback 和反向代理不工作
- 涉及文件：`nginx/nginx.conf`
- 修复：替换为正确的 nginx 配置（SPA fallback、/prod-api/ 代理到后端、gzip）

**断点 7：nginx Docker 容器无法解析后端地址**
- 严重度：阻塞
- 现象：`proxy_pass http://app:8080` 中 `app` 无法解析（后端以独立进程运行，不在 Docker 网络内）
- 修复：使用 `docker run --add-host app:192.168.100.109` 将宿主机 IP 映射到 `app`

**断点 8：部署链路不一致（源码已修复但运行旧 jar）**
- 严重度：阻塞
- 现象：源码中的 Bug 已修复，但运行的 jar 是旧版本（2026-06-21 构建）
- 解决：用户重新 `mvn package` 并部署新 jar，重启后端

**断点 9：种子数据密码不匹配**
- 严重度：阻塞
- 现象：student01 的 bcrypt hash 与 student123 不匹配
- 解决：通过 admin API `PUT /system/user/resetPwd` 重置密码

**断点 10：竞赛报名时间未到**
- 严重度：阻塞
- 现象：竞赛 400 的 applyStartTime=2026-07-01，当前时间 2026-06-28
- 解决：通过 admin API 修改 applyStartTime 为 2026-06-01

## 七、遗留问题（当前未修复）

### 1. 通知删除无归属校验（P0 安全）
- **位置**：`SysNotificationController.java` 第 99 行
- **问题**：`DELETE /system/notification/{ids}` 直接调用 `deleteByIds(ids)`，SQL 为 `DELETE FROM user_notification WHERE id IN (...)`，不含 `user_id` 过滤
- **风险**：任何登录用户可删除任意用户的通知
- **建议修复**：在 Service 层添加 userId 参数，SQL 加 `AND user_id = #{userId}` 条件（admin 角色可跳过）

### 2. XSS 存储型漏洞（P0 安全）
- **位置**：竞赛描述、队名等字段入库未转义
- **问题**：`<script>alert('xss')</script>` 原文存入 DB 并返回前端
- **风险**：可注入恶意脚本，影响所有查看该页面的用户
- **建议修复**：入库前 HTML 转义或使用白名单过滤（如 Jsoup.clean）

### 3. 草稿竞赛可创建队伍（P2 逻辑）
- **位置**：组队接口 `POST /student/team/create/{compId}`
- **问题**：未校验竞赛发布状态，草稿状态即可创建队伍
- **建议修复**：添加竞赛状态校验，仅"报名中"状态允许创建队伍

### 4. SQL 异常信息泄露（P2 安全）
- **位置**：全局异常处理器
- **问题**：超长输入触发 MySQL DataTruncation 异常时，完整 SQL、表名、列名、Mapper 路径通过响应返回
- **建议修复**：在全局异常处理器中捕获 `DataTruncation`/`DataAccessException`，返回通用错误信息

### 5. maxMembers 硬编码为 5（P3 设计）
- **位置**：`CompTeamServiceImpl.java` 第 75 行
- **问题**：`setMaxMembers(5)` 硬编码，竞赛创建时设置的 maxMembers 参数不生效
- **建议修复**：从竞赛对象中读取 maxMembers 参数

## 附录：测试过程中的环境排障经验

1. **Docker 镜像拉取失败**：daemon.json 中 registry mirrors 大多失效，精简为仅保留 `docker.1panelproxy.com` 和 `dockerproxy.com`
2. **端口 80 被占用**：Node.js 进程占用 80 端口，需先停止
3. **Redis/RabbitMQ 认证**：需设置环境变量 `SPRING_DATA_REDIS_PASSWORD`、`SPRING_RABBITMQ_USERNAME`、`SPRING_RABBITMQ_PASSWORD`
4. **多浏览器 Agent 并行冲突**：共享同一浏览器实例导致会话冲突，改为串行或 API 方式
5. **验证码自动化**：通过 `redis-cli GET "captcha_codes:<uuid>"` 获取 math 验证码答案绕过
